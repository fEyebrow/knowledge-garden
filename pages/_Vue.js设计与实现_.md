- 框架设计概览
	- 权衡的艺术
	  collapsed:: true
		- 声明式和命令式
			- 性能
				- 声明 = 找出差异 + 直接修改
				- 命令 = 直接修改
			- 可维护性
				- 声明优于命令
		- 虚拟dom与innerHtml
			- 初始化的时候是一样的
			- 更新时，虚拟dom的运算量与数据变化量相关。innerHtml和模版大小相关
		- 运行时与编译时
			- 纯运行时导致框架无法分析用户提供的内容
			- 加入编译时，可以分析模版，看那些会变，那些不变，将得到的结果给render，优化render的性能。
			- 纯编译时，则有失灵活性，用户提供的内容得遍以后才能使用。
	- 框架设计的核心要素,开发者体验
		- 提供warn
		- 通过tree shaking和构建工具预定义的常量来有选择地打包文件，精简打包后的体积。
		- 可以使用/*#__PURE__*/来显示地指定dead code
		- 提供多种格式的打包产物：IIFE，ESM，CommonJS以及给构建工具的esm-bundler.js.esm-bundler.js将__DEV__替换成process.env.NODE_ENV.
		- 提供特性开关来关闭对选项对象式API
		- 提供错误处理方案，允许用户注册错误处理函数
